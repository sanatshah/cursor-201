# Obsidian Documentation Skill

This skill helps manage and organize documentation for the web-photoshop project using Obsidian markdown files in the `docs/` folder.

## When to Use

- Use this skill when creating, updating, or organizing project documentation
- This skill is helpful for maintaining structured documentation for the web-photoshop project
- Use when you need to reference or create specifications, PRDs, or architecture documents

## Documentation Structure

The `docs/` folder is the central location for all web-photoshop project documentation. The folder should be organized with the following structure:

```
docs/
├── Specifications/
├── PRDs/
└── Architecture/
```

### Directory Descriptions

- **Specifications/**: Contains detailed technical specifications for features, tools, and functionality
- **PRDs/**: Contains Product Requirements Documents outlining product features, user stories, and requirements
- **Architecture/**: Contains architecture documentation, system design, technical decisions, and diagrams

## Instructions

- Always create documentation files in the appropriate subdirectory based on their content type
- Use markdown format (.md) for all documentation files
- Maintain clear, structured documentation following the folder organization above
- When creating new documentation, place it in the most appropriate subdirectory (Specifications/, PRDs/, or Architecture/)
- Reference existing documentation when making decisions or implementing features
